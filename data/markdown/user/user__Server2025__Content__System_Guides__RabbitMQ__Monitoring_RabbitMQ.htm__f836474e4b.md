# Monitoring RabbitMQ

RabbitMQ has many options for monitoring your deployment, both manual and automatic. While the specific values to monitor will vary based on features and the size of your Relativity instance, excessive queue depth and server health statistics are likely the most valuable metrics to monitor.

See these related pages:

- For custom automation, you can use the RabbitMQ management HTTP API. For more information on the API, see the HTTP API link at the bottom of the RabbitMQ management UI .

- For more information about general monitoring, see monitoring on the RabbitMQ website .

- For a list of third-party tools commonly used to collect RabbitMQ metrics, see monitoring tools on the RabbitMQ website .
