# Relativity User and Group Synchronization

You can use the Relativity User and Group Synchronization application to synchronize groups, users, OpenID login methods, and clients from a designated master instance to a connected duplicate instance.

- Master - refers to the Relativity instance which acts as the trusted source and contains users, groups and clients to be synchronized.

- Duplicate - refers to the Relativity instance that is the target of synchronization.

This functionality is only available for customers running at least one RelativityOne instance, as well as a second instance running either RelativityOne or Relativity 9.5.224.9 and above.

## User and Group Synchronization topics

Topic Description

Installing the User and Group Synchronization applications The User Group Synchronization application has two different RAP files, one for your Master instance and one for your duplicate instances. This topic contains installation instructions.

Configuring your environments After installing the User Group Synchronization applications, you need to configure your environments. This topic walks you through how to set up and configure your master and duplicate instances.

Working with User and Group Synchronization After installing and configuring the User and Group Synchronization application in your environments, you can sync users from your master instance to your duplicate instance, view user and group synchronization information, and monitor the status of jobs that the User Group Sync agent processes during a sync.
