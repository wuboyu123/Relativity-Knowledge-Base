# Feature Domain

## Context

This document defines the official feature domains defined for Relativity Server used by the Environment Watch product to define standard observability metrics/traces/logs, combined with alerts and health check results, that are used to group various 'signals' within top-level visualizations to provide the end user a simple way to assess the overall instance health.

## Official Domains

Domain # Name Examples

01 ADS Workspace upgrade, Application installation

02 Agents Integration Points Agent, Conversion Cache Discovery Agent

03 Analytics Structured/Conceptual/CAAT

04 Authentication and Permissions Identity Server, OAuth2

05 Billing and Licensing Relativity/Invariant license expired

06 Certificates TLS certification validation, expiration

07 Configuration Developer mode Instance Settings

08 Data Store SQL Server, Elasticsearch, Data Grid

09 Data Transfer ARM, RDC, RIP, SFU

10 Developer Platform Bootstrap Exception, Runtime Framework, EventHandlers

11 Document Transformation Productions/Conversion/PDF/Imaging

12 Eventing RabbitMQ

13 File Storage File shares, BCP share, cache location

14 Host Infrastructure CPU, Memory, Process, Network

15 Legal Hold Email notification failure

16 Mass Operations Mass edit/update/delete failure

17 Monitoring OTEL/SDK, Environment Watch

18 Processing Entity, Password Bank

19 RDO Object manager CRUD failure

20 Review Doc viewer, Case Strategy, Conversion

21 Search, Lists, and Visualizations Fluid UI, Liquid Forms, Pivot

22 Service Host Kepler service exceeded 3 retry attempts

23 Unassigned All third-party/non-tagged Alerts

24 Web IIS, custom pages deployment failure
