# Document Review Overview

This dashboard provides a high-level view of document viewer activity within Relativity's Review Interface. It tracks key performance metrics such as document load times, viewer success rates, and response latency. By analyzing these metrics, users can assess review performance, detect viewer slowdowns, and identify potential issues impacting user experience. It helps teams maintain consistent review speeds and ensure the document viewer performs efficiently across active workspaces and document types.

## Review Performance Summary

The Review Performance Summary section provides an overview of viewer performance metrics, including load time averages, slow response percentages, viewer load success rate, and active user count. Health indicators such as Avg Load Time and Slow Responses immediately signal if users are experiencing delays or errors. A breakdown by Document Type helps visualize the proportion of documents opened during review, offering insight into workload distribution (e.g., MSG, XLS, PDF files). This section enables users to confirm that viewer activity is within expected thresholds and that no performance bottlenecks are present.

## Viewer Navigation and Response Trends

The Viewer Navigation and Response Trends section tracks how users navigate documents over time and how long different viewer operations take to complete. The top visualization displays navigation counts by viewer type (e.g., native or image viewers) to highlight review activity levels and trends.

The bottom tables detail Time to Display by Navigation Type , allowing administrators to identify operations that may require optimization such as switching viewers, saving progress, or navigating between documents. Monitoring these trends helps maintain consistent reviewer productivity and ensures quick transitions between document views.

### Use Cases

Use Case Description

Monitor review performance Track document load times and latency trends to ensure fast, reliable document loading during active review sessions.

Troubleshoot viewer issues Review viewer performance metrics and navigation trends to identify affected operations and potential causes when users report slowness or load failures.

Analyze document type distribution Assess which document types (e.g., MSG, XLS, PDF) dominate review activity to plan resource allocation and confirm efficient viewer handling.

Ensure consistent user experience Track load success rates and slow-response percentages to proactively address performance issues before they affect reviewers or case teams.
